30 Phases, 3 Waves

Wave 1 — Phases 1-12

Static files, CSS, templates, Dockerfile, JS section headers, route stubs

Wave 2 — Phases 13-20

server.py deleted, service layer created, MODEL_REGISTRY, exceptions, 144 unit tests

Wave 3 — Phases 21-30

Route extraction, JS module splits, provider splits, agent.py refactor

Rule: One commit per phase. Feature parity only. Zero behavior changes.

The Test Story

The Problem

  • Phase 20: 144 unit tests added
  • After full refactor: 8 failures
  • Could have blamed external APIs. We did not.

The Fix

  • Bug 1: call_model() returning dict, not string
  • Bug 2: Copilot mock patches targeting wrong module after split
  • Bug 3: conftest import path broken in e2e tests
  • Result: 283 pass / 0 fail

What We Missed

JS — Invisible to Tests

  • redrawGrid — deleted, put nowhere
  • let currentUser — duplicated; crashed JS parser, app never started
  • renderSessionTabs + getTabDotClass — deleted, called 20+ times
  • _PROMPT_SECTION_MAP — removed, not re-added
  • 3 split modules wired nowhere (session-storage, replay, persistence)
  • Duplicate function declarations across human*.js files

Python

  • agent_scaffold.py — broken import; batch runner crashes
  • server/app.py — not directly executable (sys.path)
  • app.py.backup — 836 lines of dead code committed
  • 283 tests passed. App did not run.

The Lesson

  • Unit tests check code. They do not check whether the app loads.
  • 283 green + 0 failed ≠ "ready to PR"
  • The agent driver (Mark) caught a non-functional browser app after the refactor was declared complete
  • New rule: No PR without end-to-end browser smoke test + agent driver sign-off
